# Техническая документация "Автоматической системы точной посадки (АСТП) БВС"

## Алгоритм

1. Дрон перемещается к точке назначения, ориентируясь на ГЛоНАСС (GPS) с погрешностью дистанции
 1-3 м и набирает/снижает высоту до $\approx$ 30 м над уровнем поверхности.
2. С помощью камеры модуля АСТП, направленной ортогонально вниз, делается снимок поверхности (изображение 480x270 px / 19.54 КБ).
3. С помощью системы эхолокации модуля АСТП, направленной ортогонально вниз, составляется топографическая карта поверхности (480x270 px / 7.58 КБ или матрица из float-значений расстояний).
4. С модуля дрона на центр управления отправляется два полученных скана (собственным протоколом или отдельно по каналу передачи медиа)
5. На стороне центра управления полученные изображения коррелируются (суммируются значения пикселей)
6. По полученному коррелированному изображению с помощью машинного зрения (OpenCV) определяется положение
прямоугольника площадки, его смещения относиетльно положения и координат вершин прямоугольника и угла наклона поверхности
7. Рассчитывается смещение позиции дрона и крена по осям X/Y/Z
8. Передеается команда на перемещение дрона в виде сообщения стандартного протокола

- Тяжелые навигационные геометрические вычисления выполняются на стороне управления полетами, модуль не обязан нести на себе устройства с большой вычислительной мощностью,
что упрощает техническое сопровождения (питание, охлаждение)
- Все необходимые для корректировки посадки данные можно передать по медиа-каналу или по основному каналу в виде потока байтов ($\approx$ 7 пакетов по 8 Кб (56 Кб данных))
- Алгоритм не использует подготовленные данные о местоположении площадки или ориентировочные маркеры, расчет происходит "на лету"
- Алгоритм пригоден для условий ограниченной видимости, так как визуальные данные используются лишь для корректировки полученных данных топографии местности
- Решение может быть выполнено в виде съемного модуля, что позволит использовать решение только если имеется необходимость в точной посадке.
- Вес модуля не превышает 200г, что не составляет большой нагрузки для дрона
- Время вычисленй не превышает 50мс, задержка между прилетом на точку и посадкой определяется только задержкой передачи сигнаолв от центра управления до дрона и обратно
